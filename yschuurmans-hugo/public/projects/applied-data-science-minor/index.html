Applied Data Science Minor

After 3 years of software development I wanted to try something new. Upon discovering the Applied Data Science Minor, my choice became an easy one.

Since I had always been interested in machine learning and data wrangling, the Applied Data Science Minor appealed to me. After some research it became clear that it consisted of three primary subjects. Each subject has a strong reason to be included in the minor, and every single one is important to do research on a dataset.

Getting and Cleaning of Data

The first step of every Data Science project is to get the data you want to analyse. Often these datasets come from companies, but many datasets can also be acquired from the internet on sites like Kaggle.

After the dataset has been downloaded it is important to clean the dataset, which means removing or changing rows that might not be usable. An example of this might be a dataset of student grades where some grades are written on a 1-10 scale, some are written on an A-F scale, and others are based on descriptions such as Excellent, Good, Sufficient, or Insufficient. In this case cleaning the data means converting every value to the same system, so accurate data research can be applied over all rows at the same time.

When the data is cleaned and ready, it might be important to add more relevant information. Using the example of student grades, maybe the weather influences the mood of the students, resulting in better or worse grades. When a correlation like this seems plausible it might be interesting to also download that dataset and enrich your dataset with it. Using this combination you might discover correlations which you might not have found without the extra dataset.

Data Visualization and Exploration

After a dataset has been gathered, cleaned and optionally enriched, your dataset is ready to be used for research. The first step is to explore the dataset by visualizing it and discovering how it looks in the grand scheme of things. This part is important because if you do not explore your data, you have no idea where to look for correlations. By visualizing different aspects of your dataset you get a feel for the data, which features might have a relation with which other columns, and which features might need more enrichment before they can be used.

Machine Learning

The last step is applying the machine learning algorithms themselves, training a model with the data, and using the model to predict values. While this is the most interesting step of the whole process, it is often the shortest one. This step begins with choosing an algorithm based on the type of input, the type of data in the dataset, and the required type of output. When an algorithm has been chosen it is time to implement it and train it using the data. After the training is done you need to test the quality of the model by comparing predictions against a test set that is not used in the training. The better it predicts the data in the test set, the better the model is.

From this point you attempt to improve the model by tweaking the parameters or trying another algorithm, and once the accuracy is high enough the model is ready for real use. A finished model can be used to predict future situations in real life. Using the student grades example again, it might be possible to predict a student’s grade by their participation rates for that class compared to other classes and grades. Using this information the school might be able to prevent a failing student from failing before it is too late.
